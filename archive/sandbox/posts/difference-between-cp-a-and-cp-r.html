Should I use cp -a or cp -r when fully mirroring a directory? 3/22/2017

This comes up a surprising amount in forums and mailing lists, probably because cp's man page isn't too descriptive, so I thought it would be a good idea to break down the differences.

High Level:

cp works by copying everything in a directory to a new location. By default it skips directories and the metadata comes from the command execution and not the original file. So the date is the time of the copy and not the original time created. The owner is the user that triggers 'cp' and not the original owner and so on.

cp -r or cp --recursive adds the ability to also copy directories and is the same as cp -R on most Linux installations. The issue then, is that you still have the metadata changing from the copy. You can add -p to fix these issues, but that doesn't give you the fullest preservation options.

cp -a or cp --archive is essentially cp -r with a full suite of preservation options. It is the same as cp -dR.

Low Level:

Ignoring a lot of the extra features added to cp for legacy or edge case systems, it starts off by

cd -- -test

Easy enough once you know that trick. You can also cheat and define the path explicitly, but that isn't as fun to whip out at parties:

cd ./-test
